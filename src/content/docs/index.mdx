## The Fast Fail Principle

Every HTTP request passes through four ordered stages. A failure at any stage
immediately returns a standards-compliant error response — no downstream work
is wasted.

<CardGrid>
  <Card title="1. Negotiation" icon="approve-check">
    Content type, language, and encoding negotiation per RFC 9110 §12.5.
    Rejects unacceptable requests before any processing begins.
  </Card>
  <Card title="2. Authorization" icon="warning">
    Bearer/Basic authentication per RFC 6750 / RFC 7617. Unauthorized requests
    are rejected before touching business logic.
  </Card>
  <Card title="3. Validation" icon="pencil">
    JSON Schema body validation, query parameter parsing, and input
    sanitization. Malformed requests fail before execution.
  </Card>
  <Card title="4. Execution" icon="rocket">
    Only valid, authorized, properly-negotiated requests reach your handler.
    Responses use RFC 9457 Problem Details for errors.
  </Card>
</CardGrid>
